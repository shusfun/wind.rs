项目定位

这是一个新项目，不是给旧项目做兼容补丁。目标是以 Rust 后端 + React 前端 重构 WindsurfAPI 的核心能力，让 sub2api 后续只通过 127.0.0.1 调本项目提供的上游接口。

关键原则：不改 sub2api；不引入 Node sidecar；协议层不要闭门自研，必须参考 dwgx/WindsurfAPI 原版；前端 UX 尽量复刻原版管理台，但实现使用当前 React/Tailwind v4 工程。

长期方向

  • 后端用 Rust 承担账号池、并发控制、粘性会话、限流、真实 Windsurf LS 调用、兼容 OpenAI/Anthropic API。
  • 前端用 React 管理账号、代理、批量登录、调用日志、容量参数和账号探测。
  • 生产环境由 Rust 托管构建后的前端静态文件；开发环境前端单独跑 Vite 热更新。

RemoteApiEngine 当前状态

当前核心文件是 crates/server/src/engine/mod.rs 和 crates/server/src/engine/proto.rs。运行链路已经改为远端模板调用，不再依赖本地 Windsurf 运行时。

已实现

  • 从设置读取 remoteApiBaseUrl、remoteTemplateDir、remoteRequestTimeoutMs。
  • 内置 GetChatMessage 等请求模板，并支持外部模板目录覆盖。
  • 实现 connect-proto gzip frame 编解码和远端流式响应解析。
  • /v1/messages 已改走共享 RemoteApiEngine。
  • /admin/accounts/{id}/probe 直接用目标账号调用远端引擎，不占用全局调度槽。
  • 账号列表顶部“刷新状态”只刷新账号状态/余额，不触发批量真实模型对话，避免一键触发风控。

仍需补齐

  • 当前优先覆盖 Anthropic 文本消息；OpenAI 兼容未做。
  • 图片输入未做。
  • 工具调用完整映射未做。
  • token usage 还不是原版完整逻辑。
  • 还没有跑真实 Windsurf 官方请求验证；这是外部生产 API 调用，需要操作者确认。

关键实现判断

  • 不再保留本地运行时路线。当前方向是 Rust 后端直接请求 Windsurf 远端接口。
  • 不要重新发明协议。字段号、模板 patch 和 frame parser 以 ZephyrSail 提取资料为当前真相源。
  • 4h4g 服务器上主要靠账号池、并发控制、粘性会话、队列和限流控制容量。

远端模板与本地配置

不需要下载本地 Windsurf 运行时。后端默认使用仓库内置 protobuf 模板；如果上游协议变化，可以通过外部模板目录覆盖。

后端相关设置

设置项说明
remoteApiBaseUrl远端 Windsurf 接口地址，默认 https://server.self-serve.windsurf.com。
remoteTemplateDir外部模板目录。为空时使用仓库内置模板。
remoteRequestTimeoutMs单次远端请求超时时间，默认 120000ms。
