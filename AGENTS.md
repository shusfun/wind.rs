# AGENTS 指南

## 沟通与协作
- 所有交流、分析、解释、审查结论与交付说明必须使用简体中文。
- 不替用户做产品、架构、兼容性或取舍决策；需要取舍时，只给事实、证据、风险和可选方案，等待用户明确选择。
- 不把“理论可行”“看起来能通”“编译通过”当成交付完成。
- 用户要求接入、调用、联调、可用性验证时，必须完成真实调用测试；只有请求实际打通、返回符合预期后，才能通知用户已完成。
- 如果真实测试失败，必须说明失败的具体请求、状态码、错误信息、已排除项和下一步要验证的点。

## 工作区边界
- 默认禁止把任何文件写到当前工作区之外。
- 临时文件、日志、构建产物、测试输出统一放在工作区内，例如 `./.tmp/`、`./dist/`、`./logs/`。
- 所有路径参数在执行前都必须确认仍位于工作区内；如果路径越界，必须停止执行并说明原因。

## 高风险操作
- 删除文件或目录、批量修改、移动系统文件、修改环境变量、权限变更、数据库删除或结构变更、批量数据更新、发送敏感数据、调用生产环境 API、全局安装或卸载依赖、更新核心依赖前，必须获得用户明确确认。

## 验证要求
- 能跑测试就跑测试；单次测试建议控制在 60 秒内。
- 涉及接口联调时，必须用真实 HTTP 请求或对应客户端完成验证，记录入口、关键请求路径和结果。
- 交付时必须明确说明：已验证内容、未验证内容、剩余风险。

## 本地服务与端口硬约束
- Rust 后端开发、热更新、联调和真实验证统一只使用 `127.0.0.1:3003`。
- 禁止启动 Rust 后端到 `3004` 或其他替代端口；如果 `3003` 被旧进程占用，必须先排查并停止旧进程，再用最新代码占用 `3003`。
- 前端开发服务固定使用 `127.0.0.1:9527`，Vite 代理必须指向 `127.0.0.1:3003`。
- 验证管理端接口时，必须确认请求实际打到 `3003`；不能用其他端口验证后宣称管理台可用。
- 开发环境只允许保留必要的 `3003` 后端和 `9527` 前端服务；发现多余后端端口、旧测试服务或 sub2api 残留进程时，必须先报告并清理。

## sub2api 边界
- sub2api 是计费统计入口，默认不改源码、不启动主服务、不用它代替本项目验证。
- 如果历史测试留下 sub2api 相关进程、数据库或端口监听，必须排查清楚来源；用户要求清理时，停止对应本地测试进程。
- 本项目对接目标是让 Claude Code / sub2api 等客户端通过本项目 `3003` 暴露的接口调用成功；不能通过修改 sub2api 来绕过本项目问题。
